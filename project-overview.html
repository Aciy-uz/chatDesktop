ChatClient 项目说明

基于 Vue 3 + TypeScript 的仿微信即时通讯客户端

Vue 3 TypeScript Pinia Socket.io

做了什么

根据 chatServer 后端的 API 文档和测试页面，创建了完整的前端聊天客户端。

项目结构

src/ ├── types/ │ └── index.ts // TypeScript 类型定义（User, Message, Group 等） ├── services/ │ ├── api.ts // HTTP API 封装（30+ 接口） │ ├── rsa.ts // RSA 加密工具 │ └── socket.ts // WebSocket 服务封装 ├── stores/ │ ├── auth.ts // 认证状态（登录/登出/Token管理） │ ├── contact.ts // 联系人状态（好友/群/在线状态） │ └── chat.ts // 聊天状态（消息/会话/撤回） ├── views/ │ ├── Login.vue // 登录页（验证码 + RSA加密） │ ├── Register.vue // 注册页（头像上传） │ └── Chat.vue // 主聊天页（三栏布局） ├── components/ │ ├── Sidebar.vue // 左侧导航栏 │ ├── SessionList.vue // 会话列表 │ ├── ContactPanel.vue // 通讯录（好友/群/申请） │ └── ChatWindow.vue // 聊天窗口（消息展示+输入） ├── App.vue // 根组件 └── main.ts // 入口文件

如何启动

  1. 确保后端已启动

    先在 chatServer 目录运行 npm start，确保服务在 localhost:3000 运行

  2. 安装依赖

    在 chatClient 目录运行 npm install

  3. 启动开发服务器

    运行 npm run dev，访问 http://localhost:5173

  4. 测试功能

    注册两个账号，在不同浏览器窗口登录，测试聊天功能

# 终端 1：启动后端 cd chatServer npm start # 终端 2：启动前端 cd chatClient npm install npm run dev

注意事项

CORS 跨域

开发时前端在 5173 端口，后端在 3000 端口，存在跨域。确保后端已配置 CORS 允许跨域请求，否则 API 调用会失败。

RSA 公钥缓存

RSA 公钥在首次使用时获取并缓存。如果后端重启生成了新密钥对，需要刷新前端页面清除缓存，否则加密会失败。

文件上传大小

后端限制文件上传最大 10MB，前端也做了相同限制。如需修改，需同时调整前后端配置。

多窗口测试

测试聊天功能时，可以用 Chrome 的无痕模式或不同浏览器打开两个窗口，分别登录不同账号。

环境变量

API 地址通过 .env.development 配置，默认指向 http://localhost:3000。如需修改后端地址，创建 .env.local 文件覆盖即可。

消息撤回限制

消息撤回功能限制在发送后 2 分钟内，这是后端逻辑控制的。超过 2 分钟的撤回请求会被拒绝。

架构说明

数据流

层级 职责 文件
Views 页面级组件，组合 Components 和 Stores Login.vue, Chat.vue
Components UI 组件，接收 Props，触发 Events ChatWindow.vue, Sidebar.vue
Stores 状态管理，处理业务逻辑 auth.ts, chat.ts
Services 底层服务，HTTP/WS 通信 api.ts, socket.ts

WebSocket 事件映射

事件名 方向 说明
private_message 双向 私聊消息收发
group_message 双向 群聊消息收发
message_read 客户端 → 服务端 标记消息已读
messages_read 服务端 → 客户端 通知消息已被读取
message_recall 双向 私聊消息撤回
online_users 服务端 → 客户端 在线用户列表
user_online 服务端 → 客户端 用户上线通知
user_offline 服务端 → 客户端 用户下线通知
unread_messages 服务端 → 客户端 离线未读消息推送

迁移到 Electron

项目已为 Electron 迁移做好基础准备。以下是完整的迁移方案：

1. 安装依赖

# 安装 Electron 相关依赖 npm install electron electron-builder -D npm install vite-plugin-electron vite-plugin-electron-renderer -D

2. 创建主进程文件

// electron/main.ts import { app, BrowserWindow } from 'electron' import path from 'path' let mainWindow: BrowserWindow | null = null function createWindow() { mainWindow = new BrowserWindow({ width: 1200, height: 800, webPreferences: { preload: path.join(__dirname, 'preload.js'), contextIsolation: true, nodeIntegration: false, }, }) // 开发环境加载本地服务器 if (process.env.VITE_DEV_SERVER_URL) { mainWindow.loadURL(process.env.VITE_DEV_SERVER_URL) } else { mainWindow.loadFile(path.join(__dirname, '../dist/index.html')) } } app.whenReady().then(createWindow)

3. 配置 Vite

// vite.config.ts import electron from 'vite-plugin-electron' export default defineConfig({ plugins: [ vue(), electron({ entry: 'electron/main.ts', }), ], })

4. 添加构建脚本

// package.json { "scripts": { "dev": "vite", "build": "vite build", "electron:dev": "vite --mode electron", "electron:build": "vite build && electron-builder" }, "main": "dist-electron/main.js" }

5. Electron 特有功能

托盘图标

添加系统托盘，支持最小化到托盘、消息通知角标

本地通知

使用 Notification API 或 electron-notification 实现桌面通知

文件拖拽

支持拖拽文件到窗口直接发送

自动更新

集成 electron-updater 实现应用自动更新

本地存储

使用 electron-store 替代 localStorage 持久化数据

深链接

注册自定义协议，支持从网页直接唤起应用

迁移建议

建议先完成 Web 版本的所有功能测试，再进行 Electron 迁移。迁移时优先处理通信层（API 地址可能需要调整），然后逐步添加桌面特有功能。
